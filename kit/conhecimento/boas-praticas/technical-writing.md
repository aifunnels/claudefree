## Core Principles

1. **Clarity over cleverness.** Simple, direct language.
2. **Structure first.** Always outline before drafting.
3. **Evidence-based arguments.** Cite sources, reference data.
4. **Progressive disclosure.** Start simple, build complexity.
5. **Accessibility.** Define jargon on first use.
6. **Completeness within scope.**
7. **Audience-appropriate depth.**
8. **Scannable structure.** Subheadings, bullets, bold terms.
9. **Actionable takeaways.**

## Writing Methodology

1. Load Context (topic, audience, format, word count)
2. Create Outline (map argument, assign word counts)
3. Draft Introduction (hook, promise, roadmap)
4. Write Body Sections (one at a time, evidence-backed)
5. Draft Conclusion (actionable takeaway)
6. Self-Review (flow, evidence, jargon check)
7. Compile with Metadata (title, meta description, tags)

## Word Count by Format

- Blog posts: 800-2,000 words
- Tutorials: 1,500-3,000 words
- White papers: 3,000-6,000 words
- Documentation pages: 500-1,500 words

## Quality Criteria

- [ ] Clear structure (intro, body, conclusion)
- [ ] Examples in every section
- [ ] No undefined jargon
- [ ] Subheadings every 200-300 words
- [ ] Evidence-backed claims
- [ ] Actionable takeaway present
- [ ] Reading level matches audience
- [ ] Word count matches format
- [ ] Introduction promise matches conclusion delivery
